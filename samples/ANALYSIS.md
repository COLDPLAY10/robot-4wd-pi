# Лидар и карты SLAM: разбор проблемы и текущее состояние

## Как лидар передаёт данные

Железо: **YDLidar T-mini Plus**, UART 230400 8N1 через переходник CP210x
(порт подбирается сканом, ожидаемо `/dev/ttyUSB1`).

Управление однобайтовыми командами с префиксом `A5`:

| Команда | Действие           |
|---------|--------------------|
| `A5 60` | старт сканирования |
| `A5 65` | стоп               |
| `A5 52` | мотор              |
| `A5 50` | мотор выкл         |

Данные идут пакетами **переменной длины**:

| Байты | Поле    | Смысл                                                       |
|-------|---------|-------------------------------------------------------------|
| 0–1   | `AA 55` | заголовок                                                   |
| 2     | CT      | тип пакета; bit0=1 — стартовый пакет нового оборота (LSN=1) |
| 3     | LSN     | число сэмплов в пакете                                      |
| 4–5   | FSA     | начальный угол пакета: `(raw >> 1) / 64` градусов           |
| 6–7   | LSA     | конечный угол пакета: `(raw >> 1) / 64` градусов            |
| 8–9   | CS      | контрольная сумма: XOR всех 16-битных LE-слов пакета        |
| 10…   | сэмплы  | LSN × (2 или 3 байта)                                       |
